Day 7 Summary

Today I learned that SQL for QA is not about writing queries.

It is about validating business rules and investigating defects.

I  understands:

Data integrity validation
Account-to-user relationships
Transfer verification
Transaction investigations
Using SQL as evidence during testing

The biggest lesson from today is:

A QA engineer does not query the database because they know SQL. They query the database to prove whether the system behaved correctly.
