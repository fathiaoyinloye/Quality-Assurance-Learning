Summary
day I learned that QA testing is not just about thinking of random things that could fail. While intuition and error guessing are useful, professional QA engineers use structured techniques to design tests. I learned Equivalence Partitioning to reduce the number of test cases by grouping similar inputs, Boundary Value Analysis to focus on values where defects commonly occur, Decision Tables to systematically test combinations of conditions, State Transition Testing to verify valid and invalid state changes triggered by events, and Risk-Based Testing to prioritize the most important areas when time is limited. Through the registration form exercise, I learned that every test case should be traceable to a requirement and that structured test design produces more complete and efficient testing than relying only on instinct.
