Today I learned that a bug report is a structured document used to communicate defects to developers. A good bug report contains a clear title, reproduction steps, expected result, actual result, severity, priority, and environment information. I also learned that severity and priority are different; severity measures impact on the system while priority measures how urgently the defect should be fixed. While creating bug reports for SwiftBank, I realized that every defect must clearly show what should have happened versus what actually happened.
