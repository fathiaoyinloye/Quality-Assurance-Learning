Today I learned that a test case is a detailed document that explains how a test will be executed. A test scenario tells me what I am testing, while a test case contains the preconditions, test data, steps, expected result, and priority needed to execute the test. While creating test cases for SwiftBank Registration and Login, I learned that expected results are critical because they allow me to compare actual behavior against expected behavior. I also learned that steps are necessary so another tester can repeat the same test and obtain the same result. I noticed that I sometimes create duplicate test cases and occasionally mix different features together, so I need to improve my precision when mapping requirements to test cases.
