Today I learned that test scenarios are high-level descriptions of what should be tested, while test cases describe how testing will be performed. I learned that test scenarios are written after requirements have been clarified and understood. Requirement questions belong to requirement analysis because they help remove missing information and ambiguity, while scenarios focus on verifying system behavior. During the exercise, I created scenarios for Registration, Login, and Password Reset and learned how one requirement can produce multiple scenarios covering positive flows, negative flows, validation, security, and error handling .
